# Locations

The wiki holds the canonical description of every location: geography, history, inhabitants, its place in the world. These story location files capture something different — what a location *means within this story*. Its narrative role, how it should feel in prose, the scenes set there, any symbolic weight it carries.

Not every wiki location needs an entry here. Only those with meaningful story presence — places where significant scenes occur, that carry thematic weight, or that change in significance through the narrative.

## What an Entry Represents

One entry covers one location's story-specific notes. It answers: why is this place in the story, what does it feel like when we're there, what happens here, and does it change in meaning or physical state by the end?

## How Entries Relate to the Rest of the System

- Each entry links to the location's wiki entry via the `wiki_entry` frontmatter field (Obsidian wikilink)
- Atmosphere and prose notes here inform how the location is written in chapter prose
- The wiki location entry remains the canonical source of world-truth; this file is story-truth

## Linking Convention

Use an Obsidian wikilink in the `wiki_entry` field: `wiki_entry: "[[location-slug]]"`. Name this file using the same slug as the wiki location entry for clarity.
